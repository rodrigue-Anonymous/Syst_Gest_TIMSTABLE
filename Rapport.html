Description de la conception




1. Structure du projet :
- Le projet est organisé en différents dossiers et fichiers selon les normes de Django.
- Le dossier principal du projet contient les fichiers de configuration tels que `settings.py`, `urls.py`, etc.
- Les applications spécifiques sont créées pour gérer différentes fonctionnalités telles que l'emploi du temps, les cours, les professeurs, les salles, les groupes, etc.


2. Base de données :
- Le projet utilise MySQL comme système de gestion de base de données.
- La configuration de la base de données est définie dans le fichier `settings.py`.
- Les modèles de données sont créés pour représenter les différentes entités de l'application (plages horaires, cours, professeurs, salles, groupes, etc.).


3. Modèles de données :
- Les modèles de données sont définis dans le fichiers `models.py` de l'application emploi_temps.
- Chaque modèle représente une entité de la base de données et a des champs pour stocker les attributs de cette entité.
- Les relations entre les modèles seront définies à l'aide de clés étrangères (ForeignKey) et de clés primaires (PrimaryKey).


4. Vues et Templates :
- Les vues sont définies dans le fichier `views.py` de l'application emploi_temps.
- Chaque vue est responsable de traiter les requêtes HTTP et de renvoyer les réponses appropriées.
- Les templates HTML sont créés pour générer les pages web dynamiquement en utilisant les données fournies par les vues.


5. URLs :
- Les URL patterns sont définis dans le fichier `urls.py` de l'application et du projet.
- Les URL patterns associent les vues appropriées aux URL spécifiques.


6. Formulaires :
- Des formulaires sont créés pour permettre à l'administrateur d'ajouter et de modifier les données de l'application.
- Les formulaires aident également à valider les données soumises par les administratreur.


7. Sécurité :
- Des mesures de sécurité sont mises en place pour protéger l'application contre les attaques potentielles, telles que l'injection SQL, le cross-site scripting (XSS), etc.


8. Authentification et Autorisation :
- L'authentification des utilisateurs est gérée à l'aide du système d'authentification intégré de Django.
- Les autorisations sont définies pour contrôler l'accès aux différentes fonctionnalités de l'application.

Instructions de déploiement

1. Clonez le dépôt GitHub du projet :
git clone https://github.com/votre-nom-utilisateur/PIL1_2223_23.git

2. Assurez-vous d'avoir Python et pip installés sur votre système.

3. Créez un environnement virtuel pour isoler les dépendances du projet :
python3 -m venv myenv

4. Activez l'environnement virtuel :
- Sur Windows : myenv\Scripts\activate
- Sur macOS et Linux : source myenv/bin/activate

5. Naviguez vers le répertoire du projet : cd PIL1_2223_23

6. Installez les dépendances du projet : pip install -r requirements.txt

7. Configurez la base de données MySQL :
- Assurez-vous que MySQL est installé et en cours d'exécution sur votre système.
- Créez une base de données MySQL pour le projet.

8. Configurez les paramètres de base de données dans le fichier `settings.py` du projet Django :
- Ouvrez le fichier `settings.py` dans un éditeur de texte.
- Recherchez la section `DATABASES`.
- Mettez à jour les valeurs `NAME`, `USER`, `PASSWORD` et `HOST` avec les informations de votre base de données MySQL.

9. Effectuez les migrations pour créer les tables de base de données : python manage.py migrate

10. (Optionnel) Chargez des données initiales dans la base de données :
- Si vous avez des données de départ à charger, créez un fichier de fixtures contenant ces données (par exemple `initial_data.json`).
- Exécutez la commande suivante pour charger les fixtures : python manage.py loaddata initial_data.json

11. Lancez le serveur de développement : python manage.py runserver

12. Accédez à l'application dans votre navigateur en utilisant l'URL fournie par Django (par défaut, `http://localhost:8000/`).
Ces instructions sont une base pour le déploiement du projet Django avec une base de données MySQL.
Vous pouvez les personnaliser en fonction de vos besoins spécifiques, tels que la configuration du serveur web, la gestion des fichiers statiques, etc.
Assurez-vous d'avoir les informations de configuration appropriées pour votre environnement de déploiement, telles que
les informations de connexion à la base de données MySQL et les paramètres de sécurité.
